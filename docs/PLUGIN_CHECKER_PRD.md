# 轻量悬浮检查插件 — 功能需求文档

## 1. 背景

现有主插件具备完整的扫描、查重、入库、补全、本地导入能力，但在日常浏览时存在一个效率缺口：

**当前流程（每次需 4 步）：**

```
浏览视频页面 → 右击插件图标 → 打开 popup → 点「扫描/查重」→ 看结果
```

对于 **已经在库里、URL 完全一致的视频**，上述流程的每一步都是多余的。用户只需要 **0 步** 就知道结果——瞟一眼即可。

## 2. 产品目标

> **一句话**：页面加载时自动 URL 查重，已存在就用红点提示，不存在就什么都不显示。

- **零操作**：不需要点击任何按钮
- **零等待**：页面加载完成即出结果
- **零侵入**：不干扰页面布局、不弹出任何提示
- **极简反馈**：有红点 = 已存在；无红点 = 新视频（后续自行决定是否用主插件处理）

## 3. 使用场景

| 场景 | 行为 | 视觉反馈 |
|------|------|----------|
| 访问已入库视频的页面 | URL 完全匹配 | 🔴 红色圆点出现（持续显示） |
| 访问未收录视频的页面 | URL 在库中不存在 | **什么都不显示** |
| 不在白名单站点 | 非视频网站 | 不启动，避免误判 |

> **核心原则**：只做「已存在」的提示。不催用户操作，不弹提示框。

## 4. 功能需求

### FR-01 自动 URL 检测

| 属性 | 说明 |
|------|------|
| 触发时机 | 页面加载完成 `document_idle` |
| 检测对象 | `window.location.href`（当前页面完整 URL） |
| 后端接口 | 新增轻量 API：`GET /api/v1/videos/lookup?url=...` |
| 返回格式 | `{ exists: true/false, id: number\|null, title: string }` |
| 超时处理 | ≥ 3 秒无响应 → 不显示任何标记（静默失败） |

### FR-02 视觉反馈方案（双保险）

采用 **工具栏徽章 + 页面悬浮标签** 两种方式同时提示：

#### 2a. 工具栏徽章 (Badge)

直接显示在 Chrome 工具栏的插件图标上：

| 状态 | 显示 | 含义 |
|------|------|------|
| URL 已存在 | 🔴 红色底色 + `存` | 已在库中 |
| URL 不存在 | 🟢 绿色底色 + `新` | 未收录 |
| 加载中/超时 | 不显示 | 后端不可达 |

#### 2b. 页面悬浮标签

在页面右上角显示一个 **小型标签条**：

```
┌────────────────────────────┐
│                            │  ┌──────────┐
│                            │  │ 已存在   │  ← 🟥 红底白字
│                            │  └──────────┘
└────────────────────────────┘

┌────────────────────────────┐
│                            │  ┌──────────┐
│                            │  │ 未收录   │  ← 🟩 绿底白字
│                            │  └──────────┘
└────────────────────────────┘
```

| 属性 | 值 |
|------|------|
| 位置 | 页面右上角，距顶 12px、距右 12px |
| 尺寸 | 约 52×24px，圆角矩形 |
| 颜色 | 红底 `#ff4d4f` / 绿底 `#52c41a` |
| 文字 | 白字 `已存在` / `未收录`，12px 字体 |
| 交互 | `pointer-events: none`，不干扰页面点击 |
| 显示 | **持续显示，不淡出不消失** |

> 两个机制互补：工具栏徽章在视线余光区域，悬浮标签在页面内容区，用户扫到任意一个即可。

### FR-03 白名单机制

- 只在配置的 `source_sites` 白名单内启用
- 默认白名单：`["thisvid.com", "spankbang.com", "xvideos.com", "pornhub.com"]`
- 用户可通过选项页自行添加/删除域名
- 不在白名单的站点：**完全不获取任何权限**、不注入任何元素、不调 API

### FR-04 与主插件的关系

| 项目 | 主插件（现有 video-dedup） | 轻量插件（video-dedup-checker） |
|------|--------------------------|-------------------------------|
| 功能 | 全功能查重/入库/补全/导入 | 纯 URL 存在性提示 |
| 操作 | 需点击打开 popup | 零操作自动 |
| 反馈方式 | popup 表格 + 操作按钮 | 工具栏徽章 + 页面右上角悬浮标签 |
| 存在判定 | 多维匹配（URL/标题/时长） | 仅 URL 完全一致 |
| 配合方式 | 无红点/绿徽章时手动打开它进一步查重 | 日常浏览时自动识别，看颜色即知 |

> 两个插件**不冲突**，可同时安装。轻量插件管「看」，主插件管「操作」。

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | API 调用 ≤ 500ms（URL 索引单行查询） |
| 侵入性 | 页面内仅一个 14px 圆点，无额外 DOM 元素 |
| 安全性 | 仅访问后端本地 API（`127.0.0.1:18080`） |
| 兼容性 | Chrome MV3，不依赖 eval |
| 可维护性 | 最少文件：manifest.json + background.js + content.js |

## 6. 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| UAT-01 | 已入库页面双提示 | 打开已入库页面 → 工具栏图标出现 🔴「存」+ 页面右上角出现「已存在」红标签 |
| UAT-02 | 未收录页面双提示 | 打开未收录页面 → 工具栏图标出现 🟢「新」+ 页面右上角出现「未收录」绿标签 |
| UAT-03 | 非视频网站无干扰 | 打开百度/Google → 不注入任何元素，无徽章显示 |
| UAT-04 | 后端不可达不报错 | 断开后端 → 无任何显示，无控制台报错 |
| UAT-05 | 标签持续不消失 | 标签出现后始终可见，不淡出不隐藏 |
