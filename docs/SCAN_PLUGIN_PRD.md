# 右键批量扫描插件 — 功能需求文档

## 1. 背景

现有 checker 插件解决了「当前页面 URL 自动查重」，但对于**内容列表页**效率仍然不够：

**当前痛点：**
```
打开列表页（如分类页、搜索结果页、频道页）
  → 页面上有几十个视频链接
  → 需要一个个右键 → 新标签打开 → 等加载 → 看红点
  → 否则就得打开主插件手动逐个查重
```

需要一种方式**在列表页一次性看完所有链接的收录状态**。

## 2. 产品目标

> **一句话**：右键视频列表容器 → 自动扫描内部所有链接 → 在每个链接旁标注已收录/未收录。

## 3. 使用场景

| 场景 | 用户操作 | 结果 |
|------|----------|------|
| 浏览分类/搜索结果页 | 右键 `div.thumbs-items` → 点菜单「扫描此区域链接」 | 区域内所有链接旁出现红/绿标注 |
| 扫描完成后查看 | 逐个看标注 | 🔴 = 已收录（显示ID） 🟢 = 未收录 |
| 清除标注 | 再次右键同一区域 →「清除此区域标注」 | 标注全部移除 |
| 其他容器 | 未来可扩展（通过选择器配置） | 见后续需求 |

## 4. 功能需求

### FR-01 右键菜单

| 属性 | 说明 |
|------|------|
| 触发方式 | 鼠标右键点击页面元素 |
| 菜单显示条件 | **始终显示**（用户自行判断是否在目标容器上右键） |
| 菜单项 1 | `🔍 扫描此区域所有链接` — 触发扫描 |
| 菜单项 2 | `🧹 清除此区域标注` — 移除所有标注（仅在已有标注时显示） |
| 菜单位置 | Chrome 原生右键菜单末尾 |

> 之所以不依赖元素判断，是因为 MV3 context menu 无法根据右键目标动态显示/隐藏菜单项。用户记住在 `div.thumbs-items` 上右键即可。

### FR-02 批量扫描流程

```
用户右键目标容器
  → 选择「扫描此区域所有链接」
  → content.js 定位到右键时所在容器
  → 提取容器内所有 <a href="..."> 的 URL
  → 去重 + 过滤非视频链接
  → 批量提交 POST /api/v1/public/lookup-batch
  → 拿到结果 → 在每个链接后插入状态标注
```

**链接过滤规则：**

```
① href 以 http:// 或 https:// 开头
② 去重（相同 URL 只查一次，所有同名链接共享结果）
③ 排除 # 和 javascript: 等无意义链接
④ 最大处理数量：200 个（防止卡死）
```

### FR-03 页面内标注

扫描完成后，在每个 `<a>` 链接的**末尾**插入一个小标签：

| 状态 | 标注样式 | 含义 |
|------|----------|------|
| 已收录 | 🔴 红底白字「✔ #ID」 | 已在库中，附 ID |
| 未收录 | 🟢 绿底白字「✚」 | 未收录 |
| 查重出错 | ⬜ 灰底「⚠」 | 后端不可达 |

**标注 CSS 规范：**

```
display: inline-block（不影响原有布局）
font-size: 11px
padding: 1px 5px
border-radius: 3px
margin-left: 4px
```

> 对于纯图片链接（无文字），标注直接跟在 `<a>` 后面。

### FR-04 缓存机制

- 已查过的 URL 在本页面**不重复请求后端**
- 缓存存储在 JS 变量 `Map<url, result>` 中，页面刷新即清
- 同一个 URL 在同一页面出现多次，共享同一个标注

### FR-05 清除标注

- 右键已标注区域 → 「清除此区域标注」→ 移除所有 `.vds-scan-tag` 元素
- 清除后 `Map` 缓存不清（再次扫描直接复用）

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 单次扫描 ≤ 2 秒（100 个链接以内） |
| 侵入性 | 仅在被扫描容器内插入标注，不修改页面其他内容 |
| 安全性 | 不读取页面内任何用户输入/密码等敏感信息 |
| 可逆性 | 标注可一键清除，不留痕迹 |
| 兼容性 | Chrome MV3，不依赖 eval |

## 6. 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| UAT-01 | 右键菜单出现 | 任意页面右键 → 菜单出现「扫描此区域所有链接」 |
| UAT-02 | 扫描生效 | 在目标容器上右键扫描 → 每个链接旁出现 ✅/✚ 标注 |
| UAT-03 | 缓存生效 | 第二次扫描同一区域 → 瞬间完成（无 API 请求） |
| UAT-04 | 清除标注 | 右键「清除此区域标注」→ 标注全部消失 |
| UAT-05 | 非破坏性 | 标注不改变原有链接功能，点击仍能正常跳转 |
