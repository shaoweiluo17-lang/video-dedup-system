# 视频去重下载系统 — 产品需求文档（PRD）

## 1. 背景与问题

当前视频采集与下载流程存在三类关键痛点：

1. **重复下载严重**  
   同一视频常因链接变化、标题微调等原因被反复下载，大量浪费带宽与存储空间。

2. **视频信息分散**  
   视频元数据（来源、大小、时长、截图等）散落在不同工具和目录中，难以统一检索和管理。

3. **缺少任务追踪**  
   下载过程、截图生成、异常记录等缺少系统化的记录与回溯手段。

---

## 2. 产品目标

基于上述问题，本系统目标为：

- **在下载前完成去重校验**，大幅减少重复内容入库；
- **建立视频元数据中心**，提供统一的存储、检索与管理能力；
- **支撑批量导入、统计、截图处理与下载任务追踪**；
- **为后续 Chrome 插件接入与自动化调度提供标准化后端基础**。

---

## 3. 产品范围

### 3.1 本期范围（In Scope）

| 模块 | 说明 |
|------|------|
| 后端 API（FastAPI） | 提供 REST 接口，含 search、check、CRUD、stats、import、screenshot |
| MySQL 数据模型 | `videos`（视频主表）、`download_history`（下载历史表） |
| Redis 缓存 / 队列 | 搜索结果缓存、去重结果缓存、统计缓存、截图任务队列 |
| 去重规则与接口 | 标题标准化/拼音/时长/大小多特征分级匹配 |
| 截图任务处理 | 基于 FFmpeg 进行视频截图批处理 |
| 鉴权 | 业务接口统一使用 X-API-Key 验证 |

### 3.2 非本期范围（Out of Scope）

| 内容 | 说明 |
|------|------|
| Chrome 插件 | 后续作为采集端独立接入 |
| IDM 深度控制 | 本期仅预留适配器接口，不做真实任务状态回写 |
| 多租户权限体系 | 暂不引入用户系统 |
| 可视化管理后台（Web UI） | 本期为纯后端交付 |

## 4. 用户与场景

**目标用户**：内容采集/运营人员、视频资产管理人员

### 典型场景

| 场景 | 描述 |
|------|------|
| 下载前判重 | 用户输入 URL 或标题，系统返回是否重复及匹配详情 |
| 新增入库 | 判重通过后写入记录，自动触发下载任务与截图队列 |
| 按条件检索 | 按关键词、来源站点、时间范围查询视频列表 |
| 批量导入 | 从外部来源批量同步视频条目，输出成功/重复/失败统计 |
| 截图生成 | 下载完成后自动或主动触发截图任务，更新截图路径 |
| 统计看板 | 查询总量、总大小、总时长、待截图数量 |

## 5. 功能需求

### FR-01 去重检查

| 属性 | 说明 |
|------|------|
| 输入 | `title`、`duration_secs`、`size_mb`、`source_site`（可选） |
| 输出 | `exists`（bool）、`level`（strong/medium/weak/none）、`matches` 列表 |
| 缓存 | Redis 缓存 10 分钟（TTL=600s） |

**分级规则**：

| 等级 | 条件 |
|------|------|
| strong | 标题标准化完全相同 且 时长差 ≤ 3 秒 |
| medium | 拼音相同 且 时长差 ≤ 5 秒 且 大小偏差 ≤ 5% |
| weak | 标题模糊匹配（源字符串存在于数据库中标题） |

### FR-02 视频管理

| 操作 | 接口 |
|------|------|
| 新增 | `POST /api/v1/videos` |
| 查询列表 | `GET /api/v1/videos/search`（支持 query / page / pageSize） |
| 获取详情 | `GET /api/v1/videos/{id}` |
| 软删除 | `DELETE /api/v1/videos/{id}` |
| 更新截图 | `PUT /api/v1/videos/{id}/screenshot` |

### FR-03 批量导入

| 属性 | 说明 |
|------|------|
| 输入 | JSON 数组，每项含 url/title/size_mb/duration_secs 等 |
| 输出 | `success_count` / `duplicate_count` / `fail_count` |
| 行为 | 每条先判重，重复则计入 duplicate，否则入库 |

### FR-04 统计

| 字段 | 说明 |
|------|------|
| `total_videos` | 视频总数 |
| `total_size_mb` | 总容量（MB） |
| `total_duration_secs` | 总时长（秒） |
| `pending_screenshot` | 待截图数量 |

- 缓存：Redis TTL=3600s，变更后主动清除

### FR-05 截图任务

| 接口 | 说明 |
|------|------|
| 查询待处理 | `GET /api/v1/videos/screenshot/pending` |
| 执行处理 | `POST /api/v1/videos/screenshot/process` |
| 截图方式 | FFmpeg 截取第 1 秒关键帧 |
| 输出目录 | `SCREENSHOT_DIR`（可配置，默认 `./data/screenshots`） |

### FR-06 鉴权与健康检查

| 接口 | 鉴权 |
|------|------|
| `GET /health` | 无鉴权 |
| 所有 `/api/v1/*` | 需 Header `X-API-Key` |

## 6. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 常规查询接口 P95 < 500ms（缓存命中） |
| 可用性 | 服务可重启恢复；缓存失效不影响核心流程 |
| 安全 | API Key 校验；日志不记录敏感参数 |
| 可维护性 | MVC 分层（api/service/model/schema） |
| 可扩展性 | 预留截图任务抽象、下载器适配器接口 |

---

## 7. 验收标准（UAT）

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| UAT-01 | 去重接口 | 可正确返回 strong/medium/weak/none 四类结果 |
| UAT-02 | 新增重复视频 | 新增时若已有记录返回冲突或命中信息 |
| UAT-03 | 批量导入统计 | success/duplicate/fail 计数准确 |
| UAT-04 | 截图处理 | 处理后 `screenshot_path` 正确更新 |
| UAT-05 | 统计接口 | total_videos / total_size_mb / total_duration_secs / pending_screenshot 字段齐全 |
| UAT-06 | 鉴权 | 无 Key 访问业务接口返回 401 |
| UAT-07 | 文档完整性 | MySQL 脚本一次执行建表成功，README 可指导启动 |
